# Monday Weekly Briefing — Orchestration Spec

**Use:** Reproduces the Monday morning weekly PD briefing. Reverse-engineered from comment history on running log task `1214208955674591` between 2026-04-22 (Week 17) and 2026-04-27 (Week 18) — only two samples exist, so this spec carries some interpretation. Adjust as more briefings post.

**Trigger:** Monday morning, scheduled (`sjs-monday-pd-briefing`). Manual fallback: "run the Monday briefing."

**Output:** Comment posted to running log task `1214208955674591` in AC Brands PD + Ops Dashboard. HTML dashboard saved to outputs as `sjs-weekly-briefing-YYYY-MM-DD.html`.

---

## Architecture principle

Same as the daily recap — this is a **composition of PD skills** over a 7-day window, not a parallel tool implementation. Each PD skill encapsulates its own tool patterns. The Monday briefing orchestrates them and adds Skill 4 (Asana ↔ PLM Bridge) for the financial-signal sweep that's specific to weekly cadence.

## Prompt structure

```
Run the Monday weekly PD briefing for Sweet July Skin product development.
Today's date is [auto-resolve, should be a Monday]. Current ISO week is
[auto-resolve]. Window: past 7 days through today, plus the next 7 days for
the week-ahead view.

ORCHESTRATION

Activate the PD skills in this order. Each skill returns its findings to feed
the synthesis step. Run silently — do not narrate which skill is firing.

1. asana-pd-manager (Skill 1)
   Scope: full PD portfolio state + 7-day activity sweep.
   Output expected: RAG roll-up with project names by color, overdue task
   counts (with the heaviest-overdue cluster named), task counts due this
   coming week, supplier count in active loop, projects with recent activity,
   stale/silent projects, new projects added in the past 7 days.

2. fireflies-asana-bridge (Skill 2)
   Scope: past 7 days of meetings.
   Output expected: cross-meeting themes, decisions captured, blockers raised,
   action items still open, PLM flags from the week.

3. outlook-asana-bridge (Skill 3)
   Scope: past 7 days of emails, Inbox + Sent Items.
   Output expected: weekly triage — supplier movement, no-reply-yet items
   that crossed the 5-day window, action commitments Alvin made, urgent
   threads that didn't fully resolve.

4. outlook-plm-bridge (Skill 6)
   Scope: past 7 days of emails, Inbox + Sent Items.
   Output expected: PLM-bound activity by flow — POs sent and acknowledged,
   batch COAs received, vendor updates, formula approvals, test results,
   invoices processed, BOM/component spec changes.

5. asana-plm-bridge (Skill 4)
   Scope: PLM activity over the past 7 days.
   Output expected: PO state changes (deposits processed, ship dates updated,
   shipments received), new batches logged, vendor record updates, product
   phase changes. Financial signal in particular — invoice amounts, deposit
   confirmations, PO totals.

6. sjs-status-reporter (Skill 5)
   Format: Monday weekly briefing.
   Inputs: outputs from Skills 1, 2, 3, 4, 6.
   Output: the comment text below + the HTML dashboard.

WHAT NOT TO DO

- Do not bypass the skills and pull from tools directly.
- Do not list every project in narrative form. Names only, comma-separated,
```

## Optional richer sections

The Week 17 briefing (Apr 22) included sections that the Week 18 briefing dropped. They cost more to produce but add operational signal. Use any of these as opt-in additions when the week warrants it (heavy launch prep, post-launch retrospective, end-of-quarter):

### OVERALL POSTURE — RAG label + narrative paragraph

```
OVERALL POSTURE: 🟡 AMBER
[2-4 sentence narrative on portfolio state. Names the pressure point and the
single immovable date driving the calendar.]
```

### TOP 5 THAT MOVED LAST WEEK

```
TOP 5 THAT MOVED LAST WEEK
1. [Item with context — what changed, why it matters]
2. [Item]
3. [Item]
4. [Item]
5. [Item]
```

### TOP 5 TO WATCH THIS WEEK

```
TOP 5 TO WATCH THIS WEEK
• [Item] | Owner: [Name] | Due: [Date]
• [Item] | Owner: [Name] | Due: [Date]
```

### IMMOVABLE DATES

```
IMMOVABLE DATES
• [Date] — [Launch / milestone]
• [Date] — [Launch / milestone]
```

### OVERDUE (named items, not just count)

```
⚠️ OVERDUE ([N] items)
• [Item]: [Owner], [N days late]
• [Item]: [Owner], [N days late]
```

### STALE PROJECTS

```
STALE PROJECTS — NEED STATUS ([N])
• [Project name] ([N+ days silent / new entry])
```

### NEW PROJECTS THIS WEEK

```
NEW PROJECTS THIS WEEK ([N])
• [Project name] — [brief context]
```

If the richer format is the desired version, the daily recap and the Monday briefing diverge: daily stays compact, Monday becomes the full picture.
