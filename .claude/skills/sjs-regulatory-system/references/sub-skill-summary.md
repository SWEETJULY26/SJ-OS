# System C sub-skill summary

One-paragraph profile of each of the four System C sub-skills (regulatory-manager, claims-il-and-label-keeper, adverse-event-and-recall-reporter, regulatory-status-reporter). The router reads these at activation to decide where to forward.

## regulatory-manager (v6.3)

The System C umbrella. Owns cross-skill rollup, registrations tracker (MoCRA listings, MoCRA facility registrations, CA Prop 65, CA Fragrance and Flavor Right to Know, CA Toxic-Free Cosmetics Act, NY/WA/OR cosmetic registries, Leaping Bunny Certification, retailer-specific filings), retailer attestation cadence dashboard, Pedrero liaison surface (cross-skill view of pending/sent/overdue Pedrero work plus engagement-level work like contract renewals and scope changes), ayesha-weekly-briefing seam for founder-level signals. Intercepts Quality's `[Reg Flag Pending — regulatory-manager]` queue from SJS Quality Management and fans out (Operator-confirmed type → claims-il-and-label-keeper for IL/claim/label/attestation; adverse-event-and-recall-reporter for SAE/recall/state AE; own queue for registrations or Pedrero engagement). Operates SJS Regulatory Management Asana project (gid 1214660807386611). Read-most with HITL gates: cross-skill task creation, registration writes, attestation reminder edits, fan-out routing decisions. SOPs walked: none authored here (operates per SKN-OPS-008 and SKN-OPS-009 from sub-skills). Sample triggers: "what's open in regulatory," "any registrations renewing," "next attestation due," "what's pending with Pedrero," "monthly regulatory summary."

## claims-il-and-label-keeper (v6.1)

Owns the regulatory artifacts that wrap a SKU on both ends of life — pre-launch ingredient list (IL) review gate, sustained claim substantiation per SKU per claim, label artwork archive with IL match cross-check, retailer attestation responses (Sephora Clean+Planet Positive, Ulta Conscious Beauty, Whole Foods, Credo). Walks SKN-OPS-008 IL/Claims/Label Procedure. Receives the IL gate trigger from asana-pd-manager when Formula Tracker hits Signed Approvals (`IL Status = Pending IL Review` flips and creates inbound task). Stages every IL packet, claim sub write, attestation response, and label archive entry to Pedrero with subject prefix conventions (`[IL Review — SKU-CODE]`, `[Attestation Review — RETAILER — SKU-CODE]`, etc.). Operates inside SJS Regulatory Management (shared project with regulatory-manager; lifecycle-state-based sections: Inbound Staging → In Pedrero Review → Returned → Active / In-Effect → Renewal Window → Closed). HITL on every Pedrero send and every retailer-facing submission as separate Reg Lead gates. Pedrero return windows: default 10 business days for IL/claim/label work; tighter for retailer attestations near renewal deadlines. Sample triggers: "stage the IL on [SKU]," "log the new IL version," "what claims do we have on [SKU]," "review this new claim," "stage the [retailer] attestation," "did Pedrero sign off on the new label," "archive the [SKU] [carton/component] artwork."

## adverse-event-and-recall-reporter (v6.2)

Owns the agency side of MoCRA serious adverse events and FDA product recalls. Walks SKN-OPS-009 Reportable Events Procedure. Receives cross-flag from Quality via regulatory-manager fan-out (the upstream chain runs SJ Skin Complaint Log → complaint-and-event-handler `[Reg Flag Pending — regulatory-manager]` → regulatory-manager Job 1 routing → this skill). Drafts proposed classifications (MoCRA SAE definition criteria; 21 CFR 7 Class I/II/III for recalls), composes packets (event facts, batch context, complaint linkage, supporting docs), stages to Pedrero for binding sign-off, submits filings (FDA MedWatch portal, FDA recall reporting portal, state DOH channels for CA/NY/WA/OR). Operates SJS Reportable Events Asana project (gid 1214660834583706). Tracks statutory clocks: 15-day MoCRA SAE clock with 60%/80%/95% reminder schedule; 1-3 day Class I (hourly past 80%); ~10 day Class II; ~30 day Class III. HITL on every Pedrero send AND every agency submission as two distinct gates per filing — separate audit-trail entries. Pedrero return windows: 3 business days SAE, 1 business day Class I recall, 5 business days Class II/III/State AE. Pedrero on-call expectation for SAE and Class I recalls: same-day acknowledgment. Sample triggers: "log this SAE for FDA reporting," "what's the clock on the [SKU] SAE filing," "stage the recall report for Pedrero," "any open SAEs at FDA," "did we close the [batch] recall agency-side," "draft the FDA Class [I/II/III] report."

## regulatory-status-reporter (v6.4)

The rendering layer for System C. Generates the branded HTML regulatory dashboard at `acb-thelanding.netlify.app/regulatory-dashboard.html` from regulatory-manager's rollup. Reads from both project gids via the Netlify Function `sjs-regulatory-rollup.js` (SJS Regulatory Management + SJS Reportable Events). Renders four KPI sections: Reportable Events with statutory clock bars (green/yellow/red zones at 60%/80%/95% per SKN-OPS-009), Pedrero (cross-skill rollup of pending/sent/overdue), Retailer Attestations + Active Registrations (Sephora/Ulta/Whole Foods/Credo cadence + MoCRA/state/Leaping Bunny renewal windows), IL + Label Artwork (pre-launch flow). Hub chrome via `Hub.renderHeader/Nav/Footer` inside `<div class="page">` (per memory: NOT the site-header.html partial). On-demand regenerate + monthly auto-snapshot to `archive/regulatory-YYYY-MM.html`. Writes to two destinations: Asana attachment in SJS Regulatory Management Cross-Skill Dashboard section, and the GitHub repo at `/Users/alvinbelt/Downloads/acb-thelanding/`. Per memory: never deploys directly to Netlify — always commits and pushes to main. Sister to quality-status-reporter, sjs-status-reporter, future ops-status-reporter. Sample triggers: "regenerate regulatory dashboard," "publish the regulatory dashboard," "monthly regulatory snapshot," "branded regulatory update for [audience]."
