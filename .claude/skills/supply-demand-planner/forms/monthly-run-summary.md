# Monthly run summary

This is the canonical shape for the parent task in the Monthly Run section.
The summary lives in the task description. Bulk approval reads against this.

## Template

```
[YYYY-MM] SWEET JULY SKIN S&OP RUN
Run ID: [UUID]
Generated: [ISO timestamp]
Generated by: supply-demand-planner v3
Active channels: [DTC, UBM, Amazon]
UBM launch: 2026-06-01

— FORECAST SCOPE —
SKUs forecasted: [N]
SKU × channel × month rows generated: [N]
Plan A id: [UUID] (finished-good)
Plan B id: [UUID] (component, paired)

— TOTALS BY CHANNEL —
DTC:    [units] across [N] SKUs
UBM:    [units] across [N] SKUs
Amazon: [units] across [N] SKUs

Auto-adjustments
- [SKU] [channel] [month]: [factor] from [source] | Before [units] → After [units] | [Reject link]
- ...
(when zero adjustments fired, write a single "- None" bullet instead of the SKU list)

IMPORTANT — do not change this section's shape. The S&OP landing dashboard's "What changed this run" parser at /.netlify/functions/sjs-sop-asana anchors on a line that starts with the literal word "Auto-adjustments" (case-insensitive) and grabs the bullets that follow until the next blank line. Don't wrap the heading in em-dashes, don't add a count line or "List:" preamble, don't reorder. Bullets must use - or • or * markers.

— EXCEPTIONS PULLED FROM BULK —
[count] exceptions auto-flagged.
List (or "None"):
- [SKU] [channel] [trigger(s)] → [Asana link to exception task]
- ...

— BUY RECOMMENDATIONS —
Finished-good: [count] tasks
Component:     [count] tasks
Total spend (estimate): $[amount]
[Asana section link]

— FILLER SCHEDULE —
KDC-One:  [count] runs scheduled
Vegelabs: [count] runs scheduled
[Asana section link]

— RISK WATCH —
[count] open risks (carries between runs).
New this run:
- [risk type] [SKU/vendor]: [one-line description]
[Asana section link]

— SETTINGS CHANGES THIS RUN —
[List any sdp_* settings changes since last run, or "None"]

— ONE-PAGER —
[Link to sjs-status-reporter branded one-pager]

— BULK APPROVAL STATUS —
[ ] Forecast updates ([N] rows)
[ ] Component explosion ([N] rows)
[ ] Inventory target updates ([N] rows)
[ ] Buy recommendations staged ([N] tasks)
[ ] Filler schedule staged ([N] tasks)

Pending exceptions to resolve: [count]
(Bulk approval gates on exceptions clearing.)

Sign off → Operations commits writes via plm-assistant.
```

## Generation notes

- Counts compute at run time from the staged data
- Settings changes pull from `settings.updated_at > last_run.created_at`
- One-pager link populates after sjs-status-reporter generates the branded
  output
- The "Bulk approval status" checklist is visual; actual approval happens via
  Operations confirmation in chat with plm-assistant

## What goes outside the description

Subtasks under the Monthly Run parent task:
- One subtask per major bucket (forecast updates, component explosion,
  inventory targets, buy recs, filler schedule) for individual checkoff
- Each subtask's description carries the row-level draft data
- All subtasks close before the parent run task moves to Archive
